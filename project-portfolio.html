Live Site GitHub

About the Site

The site balances visual flair and performance. I kept the layout modular, used semantic HTML, and layered enhancements like background sparkles & a loading bar on page click. Styles and scripts are split into dedicated files for easier maintenance and iteration.

Reflections

How did I feel?

I felt proud of how cohesive and responsive the layout turned out. The mobile-first design process helped me visualize spacing and hierarchy before scaling up, and the subtle animations add personality without slowing things down.

What did I learn?

  • Mobile-first design simplifies scaling — starting small made desktop styling far more intentional.
  • Organizing reusable components like headers and cards saved time and kept consistency across pages.
  • Adding motion and spark effects is most effective when tied to purpose — not just visuals.

Key Takeaways

  • Code structure and readability make future updates much smoother.
  • Progressive enhancement ensures performance and accessibility stay strong on all devices.
  • Maintaining a clean folder layout helps when adding new project pages later.
